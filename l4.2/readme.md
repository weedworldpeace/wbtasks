## Распределённая CLI-утилита cut с кворумом и concurrency

### 1. Запуск
    git clone github.com/weedworldpeace/wbtasks
    cd l4.2
    make start
    ./main <флаги и инпут>

### 2. Описание
    Приложение разделяет данные между воркерами(для распределенного cut) и собирает результат.
### 3. Компоненты системы:

#### App (бинарник)
    Принимает инпут и флаги через stdin
    Распределяет данные по кластеру и отправляет по http
    Собирает кворум
    Пишет результат
    !!! Использует конфиг config.yaml для адресов воркеров
#### Воркеры (Порт: 8080 и дальше по конфигу)
    Принимают по http json данные, обрабатывают(cut) и отдают обратно
    !!! Поднимаются с помощью docker-compose.yml

### 4. Тестинг
    docker compose up -d
    ./main -f 1 -d " " < input.txt 

### 5. Бенчмарк с оригинальным
    python3 bench.py

    cut time: 0.0058 seconds
    ./main time: 0.0615 seconds

    cut time: 0.0407 seconds
    ./main time: 0.5800 seconds

    Программа проигрывает оригинальной в виду latency, сериализации и десериализации
